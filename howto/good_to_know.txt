# dla tych, co jeszcze nie maja zadnego commita na koncie, ani skonfigurowanego gita polecam zaczac od linkow na koncu tego krotkiego poradnika


##########
# zawsze przed rozpoczeciem kodowania warto pobrac ostatnie zmiany z repo:

git pull

# dzieki wykonaniu na poczatku powyzszej komendy unikniemy dodatkowego commita w glownym repo w postaci: "Merge branch 'master' of https://github.com/tgrez/sejm-ngram". jesli jednak juz wykonalismy jakies zmiany, to bezpieczniej jest wykorzystac: "git fetch" i nastepnie: "git merge"


##########
# aby zacommitowac zmiany musimy gitowi pokazac, ktore pliki chcemy zacommitowac. trzeba je "dodac do commita" (tak zwany stage) nastepujaca komenda:

git add <file>

# i pozniej:

git commit -m "opis naszych zbrodni wojennych"

# albo jesli chcemy dodac do commita wszystkie zmiany w plikach z wykorzystaniem opcji "-a":

git commit -a -m "opis commita"
